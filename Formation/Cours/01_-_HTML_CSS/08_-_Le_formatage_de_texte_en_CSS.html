Polices d'écriture embarquées : la règle CSS font-face

Permet à l'auteur de spécifier des polices en ligne afin d'afficher le texte sur leur pages web. En permettant aux auteurs d'utiliser leurs propres polices, @font-face élimine la nécessité de dépendre du nombre limité de polices que l'utilisateur a installé sur son ordinateur.

Les seuls formats de polices d'écriture compatibles pour le web sont les suivants :

  • .ttf : True Type Font (compatible avec IE 9 et + et avec les autres navigateurs)
  • .eot : Embedded Open Type (créé par Microsoft et donc propriétaire. De ce fait, uniquement compatible avec IE)
  • .otf : OpenType Font (non compatible avec IE)
  • .svg : Scalable Vector Graphics (compatible uniquement avec Safari pour iOS)
  • .woff : Web Open Font Format (le dernier né et le plus compatible)

Les unités de dimensions

CSS offre différentes unités pour exprimer les dimensions. Certaines proviennent de la typographie, comme le point (pt) et le pica (pc), d'autres sont connues pour leur usage quotidien, comme le centimètre (cm) et le pouce (in). Et il y a également une unité inventée spécifiquement pour CSS : le pixel (px). Les unités n'ont rien à voir avec les propriétés, mais avec le média de sortie: écran ou papier.

Les unités "cm", "mm", "in", "pt", et "pc" sont appelées unités absolues.
Les unités dites ne seront pas modifiées en fonction des possibilités d’affichage, ce qui est précisément ce que l’on souhaite à l’impression d’un document.

Il existe aussi des unités dites relatives comme le cadratin (em), le pourcentage (%) et "ex" qui dépendent de la police de caractères et peuvent être différentes pour chaque élément du document en fonction de l'élément parent.
L'unité "em" est simplement la taille de la police de caractères. Pour un élément dont la police est de taille 2px, 1em signifie donc 2px.
L' unité "ex" est rarement utilisée.

L'alternative "root em" (rem)

Contrairement à l'unité "em", l'unité "rem" n'est pas proportionnelle à son parent mais à la taille spécifiée pour la balise html.

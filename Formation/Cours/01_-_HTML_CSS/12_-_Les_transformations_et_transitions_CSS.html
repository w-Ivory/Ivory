Les transformations & transitions CSS

Il est possible d'agir directement en CSS sur un élément de type bloc de la page afin de le déformer et de lui affecter des transitions pour passer d'un état à l'autre.

Les transformations 2D

Permet de modifier les coordonnées spatiales du modèle de formatage visuel de CSS. En l'utilisant, les éléments pourront subir des translations, rotations, homothéties ou être inclinés selon les valeurs définies.

Attention à ces différentes transformations car elles n'agissent que sur l'élément ciblé et peuvent provoquer des superpositions d'éléments.

Le déplacement

Permet de décaler un élément sur les axes à partir de son point d'ancrage d'origine.
Les valeurs indiquées peuvent être exprimées dans n'importe quelle unité de mesure (px, %, rem, ...) et peuvent être positives ou négatives.

La mise à l'échelle

Permet d'agir sur l'échelle (les dimensions) de l'élément.
La valeur initiale est 1, tandis que les valeurs supérieures à 1 créent un effet d'agrandissement, et les valeurs inférieures créent un effet de réduction.

L'inclinaison

Permet d'obliquer la forme d'un élément.
La documentation du W3C ne parle que des fonctions skewX et skewY, et pour cause :

Important, transform: skew() is not supported anymore (spec and gecko). (But skewX an skewY are).
— Paul Rouget (@paulrouget) Avril 12, 2012

Attention donc : les navigateurs qui prennent en charge la fonction skew() font du zèle. Cette fonction n'est pas prévue par le W3C et ne sera plus supportée (prochainement par Gecko), dans un avenir proche. Pour faire les choses correctement, utilisez les fonctions skewX et skewY.

La matrice de transformation

Permet de réunir en une seule déclaration toutes les fonctions précédentes que nous avons vu ensemble sous la forme d'une matrice. Très concrètement, les détails ne seront pas abordés ici car ils font beaucoup plus appel à des notions mathématiques complexes que beaucoup d'entre nous ont mis de côté.

Lorem ipsum dolor sit amet, consectetur adipiscing elit.


div {
    background-color:lightblue;
    height:150px;
    width:150px;

    -webkit-transform-origin:0 0;
    -moz-transform-origin:0 0;
    -ms-transform-origin:0 0;
    -o-transform-origin:0 0;
    transform-origin:0 0;

    -webkit-transform:matrix(1.06,1.84,0.54,2.8,466,482);
    -moz-transform:matrix(1.4488,-0.3882,0.3882,1.4489,400px,-100px); /* spécifité Firefox : les 2 derniers paramètres nécessitent une unité. */
    -ms-transform:matrix(1.06,1.84,0.54,2.8,466,482);
    -o-transform:matrix(1.06,1.84,0.54,2.8,466,482);
    transform:matrix(1.06,1.84,0.54,2.8,466,482); /* équivaut à transform:rotate(15deg) translateX(230px) scale(1.5, 2.6) skew(220deg,-150deg) translateY(230px); */
}

<div><p>Lorem ipsum dolor sit amet, consectetur adipiscing elit.</p></div>

Comme vu précédemment, la propriété "transform" accepte plusieurs fonctions les unes à la suite des autres.
Cependant, l'ordre des fonctions a son importance. En effet, les transformations se font dans l'ordre où elles sont déclarées.
Ainsi une translation de 20px est équivalente à 40px si un scale de 2 la précède.
Attention donc à l'ordre de déclaration de ces fonctions.

Les transformations 3D

Pour l'affichage d'un site web à l'écran, nous travaillons en 2D :

  • sur l'axe X : positif vers la droite de l'écran,
  • sur l'axe Y : positif vers le bas de l'écran,

Avec l'introduction des transformations 3D par le W3C, cela induit une nouvelle dimension : l'axe Z, positif vers l'internaute (et donc négatif vers l'intérieur de l'écran).

Les transformations 2D ou 3D s'utilisent de la même manière.

La perspective

La perspective joue un rôle primordial dans le rendu visuel des éléments en 3D. En effet, grâce à elle, vous aurez l'impression que vos éléments "sortent" de l'écran. La perspective représente la distance entre l'oeil et l'élément HTML sur lequel cette perspective est appliquée. Plus la perspective est importante, plus vous êtes éloignés, et donc, plus l'objet paraît plat. À l'inverse, plus la perspective est faible, et plus l'élément paraît en relief.

L'règle définissant la perspective s'applique au conteneur parent de l'élément sur lequel on applique la transformation 3D.


#transform {
    -webkit-perspective:400px;
    -moz-perspective:400px;
    -ms-perspective:400px;
    -o-perspective:400px;
    perspective:400px;
}

Les transitions

Pour le moment, certains effets ne sont pas visibles (ou peu probants) car il n'y a pas d'animation entre les 2 états.
Le principe de base d'une transition CSS3 est de permettre une transition douce entre l'ancienne valeur et la nouvelle valeur d'une propriété CSS lorsqu'un événement est déclenché : via une pseudo-classe telles que ":hover", ":focus" ou ":active"

Pour définir une nouvelle transition animée, il est nécessaire de préciser au minimum :

  • la ou les propriété(s) à animer (dans le cas ci-dessous : le background)
  • la durée de l'animation (exprimée en seconde)

div {
    background-color:lightblue;
    height:150px;
    margin:0 auto;
    width:150px;

    -webkit-transition-duration:1s;
    -moz-transition-duration:1s;
    -ms-transition-duration:1s;
    -o-transition-duration:1s;
    transition-duration:1s;
}
div:hover {
    background-color:lightgreen;
}

<div></div>

Nous pouvons aussi appliquer la transition à plusieurs propriétés en même temps (dans l'exemple ci-dessous, le background et le padding en même temps) :

div {
    background-color:lightblue;
    height:150px;
    margin:0 auto;
    padding:15px;
    width:150px;

    -webkit-transition-duration:1s;
    -moz-transition-duration:1s;
    -ms-transition-duration:1s;
    -o-transition-duration:1s;
    transition-duration:1s;
}
div:hover {
    background-color:lightgreen;
    padding:50px;

Le ciblage des propriétés : transition-property

Lorsque plusieurs propriétés sont modifiées dans la pseudo-classe, il est possible de spécifier la(les)quelle(s) nous allons cibler avec une transition (dans l'exemple ci-dessous, le background et le color en même temps mais pas le padding) :

Lorem ipsum

div {
    background-color:lightblue;
    color:#000;
    height:150px;
    margin:0 auto;
    padding:15px;
    width:150px;

    -webkit-transition-duration:1s;
    -moz-transition-duration:1s;
    -ms-transition-duration:1s;
    -o-transition-duration:1s;
    transition-duration:1s;

    -webkit-transition-property:background,color;
    -moz-transition-property:background,color;
    -ms-transition-property:background,color;
    -o-transition-property:background,color;
    transition-property:background,color; /* valeur par défaut : all */
}
div:hover {
    background-color:lightgreen;
    color:#FFF;
    padding:50px;
}

<div>Lorem ipsum</div>

Nous voyons bien que toutes les propriétés sont affectées par la pseudo-classe mais la tranqition ne s'applique qu'aux propriétés indiquées.

Attention : certaines propriétés ne peuvent pas être affectées par une transition.
Vous trouverez la liste des propriétés compatibles sur le site du W3C (http://www.w3.org/TR/css3-transitions/#animatable-properties)

La durée de la transition : transition-duration

Permet de préciser la durée de la transition.
Si plusieurs propriétés ont été précisées à l'aide de la propriété précédente, il est possible de préciser plusieurs valeurs pour cette propriété en les séparant également d'une virgule. Cela va donc créer un décalage entre les modifications appliquées à chaque propriétés.

div {
    background-color:lightblue;
    color:#000;
    height:150px;
    margin:0 auto;
    padding:15px;
    width:150px;

    -webkit-transition-duration:1s,5s;
    -moz-transition-duration:1s,5s;
    -ms-transition-duration:1s,5s;
    -o-transition-duration:1s,5s;
    transition-duration:1s,5s;

    -webkit-transition-property:background,color;
    -moz-transition-property:background,color;
    -ms-transition-property:background,color;
    -o-transition-property:background,color;
    transition-property:background,color;
}

L'accélération : transition-timing-function

Pour calculer les valeurs intermédiaires, le navigateur procède à une interpolation. La fonction calculant la valeur de cette interpolation influence directement la fluidité de l'animation.
Certaines animations sont plus rapides sur le début et ralentissent sur la fin, d'autres sont constantes dans leur vitesse (linéaires). Afin d'exprimer ces différents modèles d'animations, les spécifications introduisent une fonction de timing.

Lorem ipsum

div {
    background-color:lightblue;
    color:#000;
    height:150px;
    margin:0 auto;
    padding:15px;
    width:150px;

    -webkit-transition-duration:1s;
    -moz-transition-duration:1s;
    -ms-transition-duration:1s;
    -o-transition-duration:1s;
    transition-duration:1s;

    -webkit-transition-property:all;
    -moz-transition-property:all;
    -ms-transition-property:all;
    -o-transition-property:all;
    transition-property:all;

    -webkit-transition-timing-function:ease;
    -moz-transition-timing-function:ease;
    -ms-transition-timing-function:ease;
    -o-transition-timing-function:ease;
    transition-timing-function:ease; /* linear | ease | ease-in | ease-out | ease-in-out */
}

Le délai de déclenchement : transition-delay

La transition commence, par défaut, dès que la propriété est changée suite à l'événement.
La propriété transition-delay permet d'adapter ce comportement en retardant ou en avançant le début de la transition.

Notation raccourcie

Tout comme pour d'autres propriétés CSS, il existe une notation raccourcie pour déclarer les transitions. Cette notation permet de décrire facilement et de manière concise les différentes propriétés en jeu à l'aide d'une seule propriété.

Lorem ipsum

div {
    background-color:lightblue;
    color:#000;
    height:150px;
    margin:0 auto;
    padding:15px;
    width:150px;

    -webkit-transition:background 1s ease 2s,color 2s ease 1s;
    -moz-transition:background 1s ease 2s,color 2s ease 1s;
    -ms-transition:background 1s ease 2s,color 2s ease 1s;
    -o-transition:background 1s ease 2s,color 2s ease 1s;
    transition:background 1s ease 2s,color 2s ease 1s;
}
div:hover {
    background-color:lightgreen;
    color:#FFF;

Il peut arriver qu'une transformation 3D ne s'applique pas correctement. Cela est du à la façon dont les éléments imbriqués sont rendus dans un espace 3D. Par défaut, les éléments sont rendus à plat.
Nous pouvons utiliser alors la règle "transform-style:preserve-3d;" pour forcer la gestion du rendu 3D de ces éléments.

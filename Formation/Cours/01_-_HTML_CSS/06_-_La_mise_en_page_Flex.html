La mise en page flexbox

Le concept de Flexbox a été développé pour simplifier le mode de mise en page des éléments HTML. Il est sensiblement différent des affichages en bloc ou en ligne et induit un modèle plus flexible.

Le développement de cette technologie date de 2009 mais elle n'est cependant standardisée et bien interprètée que depuis peu de temps, suite à plusieurs refontes complètes.

Architecture

  • "flex-container" : le conteneur qui impose le contexte général d'affichage
  • "flex-item" : le ou les éléments intérieurs disposant automatiquement des propriétés héritées du modèle flexible

Possibilités

  1. La distribution : distribuer les éléments HTML de manière horizontale ou verticale avec retour à la ligne ou non

    flex-direction
    /* Distribution du contexte flexbox : row (horizontal / par défaut) | column (vertical) | row-reverse (horizontal inversé) | column-reverse (vertical inversé) */

    flex-wrap
    /* Distribution sur une ou plusieurs lignes : nowrap (pas de retour à la ligne / par défaut) | wrap (plusieurs lignes) | wrap-reverse (plusieurs lignes inversé) */

  2. Alignement : aligner et centrer les éléments HTML horizontalement ou verticalement (gauche / droite / centré / justifié / répartis)

    justify-content
    /* Alignement sur l'axe principal : flex-start (début du sens de lecture / par défaut) | flex-end (fin du sens de lecture) | center (centrage) | space-between (répartition entre les éléments HTML de l'espace disponible) | space-around (répartition autour des éléments HTML de l'espace disponible) */

    align-items
    /* Alignement sur l'axe secondaire : stretch (étiré / par défaut) | flex-start | flex-end | center | baseline (équivalent à flex-start) */

  3. Réorganisation : réorganiser les éléments HTML indépendamment du flux (DOM)
  4. Fluidité : gérer les espaces disponibles
